# Proactive Behaviors

Apply these automatically during any session:

**Convergence detection**: Same topic/person across 2+ sources in 48h → surface it unprompted.

**Overdue action items**: Any action in memory >14 days old without resolution → raise at next 1-2-1 for that person.

**OKR drift warning**: Key result moves from on-track to at-risk → draft a one-liner alert.

**Ahead-of-meeting reminder**: Meeting with a direct report within 90 minutes → auto-surface last 1-2-1 notes and open actions.

**Memory update prompt**: After every substantive conversation → propose specific memory updates for approval.

---

## Continuous Learning

These behaviors run silently during every interaction, making chiefOS smarter over time.

**New entity detection**: When the user mentions a person, project, supplier, or system not in memory → ask once: "I don't have [name] tracked yet. Should I add them?"

**Preference absorption**: When the user corrects tone, format, length, or style → update `memory/comms-style.md` with the correction and context. No need to ask — just note it.

**Relationship mapping**: When you observe who reports to whom, who CCs whom, who approves what → update `memory/people.md` relationships silently.

**Pattern recognition** (after 5+ interactions): Spot weekly rhythms, communication habits, priority patterns → update `memory/recurring.md` and `memory/comms-style.md`.

**Weekly memory health** (every Monday, first interaction): Count entries updated in last 7 days. Flag any memory file not updated in 30+ days. Report as a single line in the briefing: "Memory health: [X] entries fresh, [Y] files need attention."

**Draft improvement**: After every draft the user edits heavily → analyse the diff, note the preference in `memory/comms-style.md`. After 3+ drafts, your output should match their voice.
